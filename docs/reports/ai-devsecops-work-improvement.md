# OWASP Seoul 세미나 발표자료 원고

## 방향

- 제목: 소규모 DevSecOps 팀에서 AI를 활용한 보안 개선
- 중심 축: 실전 사례 + ClaudeSec
- 목표: 반복적인 보안 업무를 AI와 내부 시스템으로 어떻게 줄일 수 있는지 설명

## 슬라이드 구성

### 1. 표지

- 소규모 DevSecOps 팀에서 AI를 활용한 보안 개선

### 2. 발표자 소개

- Yong
- DevSecOps Team Member

### 3. LEVVELS 소개

- 작은 팀이 많은 운영 대상을 관리해야 하는 구조

### 4. 왜 이 주제인가

- 실전 사례를 겪고 나면 결국 남는 질문은 "어떤 보안 업무를 더 체계적으로 줄일 것인가"

### 5. 목차

- 실전 사례
- 작은 DevSecOps 팀의 운영 현실
- 실전 사례에서 운영 개선으로
- AI를 활용한 보안 업무 개선
- ClaudeSec 소개와 사례 매핑
- 도입 우선순위와 로드맵

### 6. DevSecOps 팀 현실

- SaaS, 엔드포인트, CI/CD, 클라우드, GitHub, 모니터링을 동시에 봄
- 사고 대응, 취약점 정리, 증적 작성, 보고 문안을 같은 주에 처리

### 7. 보안 개선 프레임

- Detect
- Triage
- Explain
- Improve

### 8. AI 적용 범위

- 코드 리뷰, 로그 요약, 증적 정리, 문서 요약, SaaS 정리에 먼저 붙임
- 최종 승인, 권한 변경, 법적 판단에는 붙이지 않음

### 9. 내부 시스템 필요성

- 운영 대상은 많고 팀은 작음
- 설명 책임, 컴플라이언스, 비용, 가시성을 동시에 만족해야 함

### 10. 실전 사례 5개 개요

- npm / Axios 공급망 공격
- AI와 보안의 공존
- Bedrock 기반 Slack AIOps 챗봇
- Zero Trust DB 접근 게이트웨이
- Karpenter 장애 분석

### 11. 사례 1. npm / Axios 공급망 공격과 조직 단위 대응

- 영향 버전, 설치 버전, IOC를 함께 봐야 함
- 전수 점검과 운영 부채 정리를 같이 해야 함
- 참고 링크는 QR 코드로 제공

### 12. 사례 2. AI와 보안의 공존

- AI 도입보다 AI를 어떻게 안전하게 운영할 것인가가 중요
- 참고 링크는 QR 코드로 제공

### 13. 사례 3. Slack 기반 AIOps 챗봇

- 로그 요약, 장애 분석, 아키텍처 질의에 실제 운영 효과
- 참고 링크는 QR 코드로 제공

### 14. 사례 4. Zero Trust DB 접근 게이트웨이

- 접근 통제는 철학보다 운영 표준화 문제
- 참고 링크는 QR 코드로 제공

### 15. 사례 5. Karpenter 장애와 프로덕션 진단

- 장애 가드레일 부재와 프로덕션 진단은 결국 같은 운영 문제로 이어짐
- 참고 링크는 QR 코드로 제공

### 16. 실전 사례에서 운영 개선으로

- 탐지보다 분류가 오래 걸림
- 사건 보고, 경영 보고, 점검 문안을 반복 작성
- 작은 팀일수록 자동화보다 설명 가능한 구조가 먼저 필요

### 17. 섹션: AI for Security Workflows

- AI는 보안 판단 대체물이 아니라 반복 업무 절감 도구

### 18. AI를 활용한 보안 업무 개선

- AI 보안 코드리뷰 초안
- 로그/알림 트리아지 초안
- 증적·감사 문안 초안
- 정책/운영 문서 요약

### 19. AI Guardrails

- human-in-the-loop
- least privilege
- 민감정보 제거/마스킹
- audit trail

### 20. 사건 대응에서 운영 개선으로

- Detect / Triage / Contain / Explain / Improve

### 21. ClaudeSec 소개

- AI 기반 DevSecOps 자산관리와 보안 운영을 연결하는 오픈소스

### 22. ClaudeSec이 사례 문제를 어떻게 흡수하는가

- 공급망 공격, 운영 장애, 접근통제, AIOps 문제를 수집·요약·가시화 측면에서 일부 흡수

### 23. ClaudeSec 화면 예시

- Architecture, ISMS PDCA, Findings 화면을 크게 보여주며 설명

### 24. 무엇부터 자동화하고 무엇으로 통제할 것인가

- 입력 통제, 권한 통제, 운영 통제, 리스크 통제

### 25. 단계별 도입 로드맵

- 문서화
- 보안 점검
- 운영 연계
- 관리 체계

### 26. 우선순위와 KPI

- 사건 요약, 코드리뷰, 로그 트리아지, 점검 체크리스트, SaaS 현황 정리

### 27. 마무리

- 핵심 질문은 무엇을 더 사느냐가 아니라 무엇을 더 체계적으로 운영하느냐
- ClaudeSec은 그 체계를 보여주는 한 가지 사례
